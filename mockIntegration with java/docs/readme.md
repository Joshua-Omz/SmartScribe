SmartMRS Mock Integration Contract (Source of Truth)

Project: SmartScribe

This document defines the strict integration boundary between the SmartScribe Go Backend and the Mock SmartMRS Java Server (Spring Boot).

1. Base URL & Environment

Local Development Base URL: http://localhost:8081 (Assuming the Java dev runs Spring Boot on its default alternative port, since Go will use 8080).

2. The Primary Endpoint: Ingesting Clinical Notes

Request Specifications (Sent by Go to Java)

Endpoint: POST /api/patient/notes

Content-Type: application/json

Authentication: (Mocked for hackathon) The Go server will send a standard Bearer token in the Authorization header: Bearer mock-hackathon-token.

Expected Payload Structure (JSON)

The Java server must be prepared to receive this exact JSON structure, representing the final, parsed SOAP data generated by the Go/AI pipeline.

Example Request Body from Go:

{
  "patient_id": "P-12345",  // Hardcoded for the hackathon
  "provider_id": "DR-987",  // Hardcoded for the hackathon
  "timestamp": "2026-04-10T19:00:00Z", // ISO-8601 format
  "clinical_data": {
    "subjective": "Reports headache.",
    "objective": "Mild fever of 101 degrees.",
    "assessment": "Fever and headache.",
    "plan": "Prescribed paracetamol."
  }
}


3. Response Specifications (Sent by Java back to Go)

The Go backend relies on these responses to confirm the data was securely persisted in the (mock) SmartMRS database.

Success Response (HTTP 201 Created)

When the Java server successfully validates the token and saves the record.

{
  "status": "success",
  "message": "Clinical note successfully synced to SmartMRS.",
  "record_id": "REC-889900" // A mock ID generated by the Java backend
}


Error Responses

HTTP 401 Unauthorized: Sent if the Authorization header is missing or incorrect.

HTTP 400 Bad Request: Sent if the JSON payload is missing required fields (e.g., patient_id).

4. Implementation Notes for the Java Dev (Spring Boot)

Framework Standard: According to the official Spring Boot documentation, use @RestController and @PostMapping("/api/patient/notes").

Data Binding: Use the @RequestBody annotation to bind the incoming JSON to a Java DTO (Data Transfer Object) class that mirrors the expected payload structure above.

Security (Bonus points for pitch): Even though it is a mock, explicitly logging that the token was validated and the data was "encrypted at rest" will sound excellent when presenting the NDPR/HIPAA compliance aspect of the project.
